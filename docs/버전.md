## 1.12.2

- **팝업 설정 변경이 커스텀 정렬을 풀어버리던 문제 해결(상호 영향 제거):** 기존엔 팝업의 모든 설정 핸들러가 `chrome.tabs.reload()`로 쿠팡 탭을 통째로 새로고침했고, 커스텀 정렬 활성 상태는 메모리 변수라 리로드 시 무조건 소실됐다(복원도 메모리 플래그 의존이라 안 됨). 정렬과 무관한 설정(키워드 필터·광고요소 제거·검색 개수)을 만져도 정렬이 풀리는 부수효과.
  - **활성 정렬 상태 영속화:** 적용된 정렬 종류와 대상 검색어를 `chrome.storage.local`(`craActiveSort`, `craSortQuery`)에 저장. 리로드/네비게이션 후 `restore*Sort()`가 이 값을 읽어 **같은 검색어일 때만** 자동 재적용. 새 검색(검색어 변경)·정렬 버튼 끄기·마스터 끄기 시 초기화.
  - **팝업 불필요 리로드 제거:** 정렬 3종·광고요소 제거·키워드 필터·검색 개수 핸들러의 `chrome.tabs.reload()` 삭제. 대신 `content.js`가 `storage.onChanged`로 라이브 반영(정렬 토글=버튼 주입/제거, 검색 개수=해당 탭에서만 `setListSizeFromAddon` 적용). 광고요소·키워드는 이미 라이브였음. 마스터 켜기/끄기만 리로드 유지.
  - **정렬 기능 끄기 시:** 해당 정렬 버튼 제거. 그 정렬이 활성 중이었으면 원래 순서 복구를 위해 리로드(활성 상태는 미리 초기화해 재적용 방지). 비활성 정렬을 꺼도 리로드 없음.
  - **효과:** 정렬을 켠 채로 다른 설정을 바꿔도 정렬 유지, 검색 개수 변경 후에도 정렬 복원, 백그라운드 탭 불필요 리로드 감소.
  - **(핫픽스) 초기화 중단 버그:** 위 작업에서 `initSortAndSizeSync()`를 최상단에서 동기 호출했는데 참조 변수 `let sortSizeChangeBound`가 호출부 아래에 선언돼 TDZ `ReferenceError`가 발생, 콘텐츠 스크립트 초기화 전체가 중단돼 "설정은 켜져 있는데 아무것도 적용 안 됨" 증상이 났다. 선언을 파일 상단(정렬 플래그 옆)으로 이동해 해결. CDP로 확장 로드가 막힌 환경이라, mock `chrome`(비동기 storage) 주입으로 content.js를 실제 페이지에서 실행해 정렬 3종 버튼·키워드 필터·스타일 주입이 모두 적용됨을 검증.

- **정렬 3종(단위가격순/할인율순/낮은·높은가격순) 전면 복구:** 쿠팡이 검색 결과 상품 마크업을 구 CSS 모듈 클래스(`UnitPrice_unitPrice__…`, `PriceInfo_discountRate__…`, `Price_priceValue__…`)에서 `fw-*` 유틸리티/무클래스 마크업으로 교체하면서, 고정 해시 클래스로 가격·단위가·할인율을 찾던 로직이 전부 매칭 실패 → 단위가가 표기된 항목도 "단위가 미제공"으로 뜨고 정렬 불가. CDP로 실제 DOM을 실측(단위가 60/60·가격 60/60·할인 39/60 확인)한 뒤, **클래스가 아닌 텍스트 패턴 기반 추출**로 전환.
  - 단위가: 리프 요소 중 `(100g당 516원)` 패턴 매칭. 기준량이 달라도 비교되도록 `금액/기준량`(1단위당 가격)으로 정규화한 `value`를 저장해 정렬 기준으로 사용(기존 문자열 숫자 이어붙이기 방식 제거).
  - 판매가: `^[\d,]+원$` 형태 리프 중 취소선(`<del>` 원가)·단위가·적립 문구를 제외한 첫 요소.
  - 할인율: `^\d+%$` 형태 리프.
  - 레거시 셀렉터는 폴백으로 우선 시도 후 실패 시 텍스트 파싱.

- (참고) v4 저장소 필드 `craPresets`/`craActivePresetId`는 미사용으로 남김(무해). 요소 메타(`name`/`category`/`enabled`)와 분류·재명명·표시 토글은 1.5.0 그대로 유지.
