# 버전

`extension/manifest.json`과 맞추는 SemVer 릴리스 기록이다.

2026-07-21에 **이력을 SemVer 의미에 맞게 다시 매겼다.** 저장 키를 마이그레이션 없이 바꾼 지점(구 문서 `1.16.0`)을 메이저 **2.0.0**으로 두고, 출시되지 않았던 세분 패치·실험 번호는 묶었다. Chrome 확장 업데이트는 새 번호가 이전 설치(`1.16.x`·`2.2.2` 포함)보다 크면 되므로 현재 버전은 **2.2.3**이다. 설정 스키마는 **10**. 구 번호 대응은 맨 아래 표를 본다.

각 항목은 **그 버전에서 일어난 변화**다. 폐기·호환성은 [`업그레이드.md`](업그레이드.md)를 함께 본다.

## 2.2.3

- 이용약관의 기능명을 「검색 개수 고정」으로 맞추고, 소개 사이트·숏폼·OG 카피를 「쿠팡용 Chrome 확장」으로 통일했다.
- 잠깐 도입했던 스키마 1–4 연속화(2.2.2)는 쓰지 않는다. `settingsVersion`은 **10**이다.
- 2.2.2를 받은 설치에 남을 수 있는 스키마 `3`/`4`는 업데이트 시 10으로 올리되, 체크=표시 의미는 다시 반전하지 않는다.
- 매니페스트를 2.2.3으로 올려 2.2.2 설치도 Chrome 업데이트 경로를 유지한다.

## 2.2.2

- 설정 스키마 번호를 연속 1–4로 바꾸려 했으나 곧 철회했다. 현행 저장 라벨은 다시 **10**이다.

## 2.2.1

- 콘텐츠 스크립트 공통 로직을 `shared-start.js`로 분리해 early/idle이 검색 개수 리다이렉트·표시 숨김 CSS를 한곳에서 쓰도록 했다.
- DOM·URL 변경 재적용을 `page-runtime.js`, `storage.onChanged` 반영을 `settings-bridge.js`로 모으고 `sort.js` 등 기능 모듈을 슬림하게 정리했다.
- `document_start` / `document_idle` 로드 순서에 `shared-start` → … → `page-runtime` → `settings-bridge` → `boot`를 반영했다.
- 확장 회귀 테스트를 `tests/extension/`에서 `tools/tests/extension/`으로 옮기고, `tools/alt-inspect` 주입 목록을 맞췄다.
- 설정 스키마는 10을 유지한다. 사용자 설정 마이그레이션은 없다.
- 개인정보처리방침의 기능 on/off 항목에서 제거된 전역 「알뜰이 사용 여부」 표현을 정리했다.
- 릴리스 이력·문서 버전 번호를 이 문서의 SemVer 체계로 재정리했다. (동작은 재정리 직전과 같다.)

## 2.2.0

- 확장 패키지를 루트에서 `extension/`으로 옮기고, Load unpacked·문서·법적 고지 생성 경로를 맞췄다.
- 팝업 전역 마스터 스위치(`altEnabled`)와 꺼둘 때 쓰던 `lastPreset` 복원·회색 잠금 UI를 제거했다. 기능은 각자 토글만으로 동작한다.
- 설정 스키마를 10으로 올리고, 업데이트 시 저장소에서 `altEnabled`·`lastPreset`을 정리한다.
- 콘텐츠 스크립트를 기능별 파일로 나누고, 검색 페이지 커스텀 UI는 짧은 reconcile(약 30ms + MutationObserver)로 빠진 것만 보강한다. 쿠팡 SPA DOM 교체 시 커스텀이 잠깐 사라졌다가 다시 붙을 수 있다.
- 표시 항목 조정으로 숨긴 상품은 정렬 순번에 넣지 않도록 셀렉터 매칭·계산 스타일 판정을 보강했다.
- 검색 개수 고정은 `document_start`에서 URL을 맞추고, 동일 대상 `replace` 중복을 막으며, 라디오는 시각만 맞춘다.
- 사용하지 않는 프레임 묶기 스케줄러(`A.schedule`)를 제거했다.
- 팝업은 세부·의견 화면 복원, 의견 목록 높이, 개수 고정 off 시에도 선택 개수 유지를 다듬었다.
- 공개 README·소개 사이트·방법론·업그레이드·출처 문서의 군살을 줄이고, 코드 설명은 주석 대신 `docs/방법론.md`에만 둔다.
- 법적 고지 서두의 「예외적으로 한국어」문구를 제거하고, 한국어 문서 기본 규칙을 따르도록 맞췄다.
- 확장 계약·순수 로직 회귀 테스트(`tests/extension/`)를 추가했다. (2.2.1에서 `tools/tests/extension/`으로 이동)

## 2.1.0

- 페이지 표시 항목 조정을 체크=표시로 바꿨다. 기능을 켜면 기본으로 모두 표시되고, 체크를 해제한 항목만 숨긴다.
- 설정 스키마를 9로 올리고, 기존 설치의 `altPresetOff` 의미를 업데이트 시 변환해 보이는 항목 집합을 유지한다.

## 2.0.4

- 의견 제출 전에 본문·확장 버전의 저장·공개 가능성, 미국 소재 서비스 처리, 개인정보 입력 금지를 안내한다.
- 개인정보처리방침에서 목록 조회(GET)와 제출(POST)을 구분하고, 의견 최대 2년·요청 제한 식별값 7일의 보존 정책을 명시했다. 의견 API는 요청 시 730일이 지난 의견을 삭제한다.
- 개인정보처리방침과 이용약관을 `extension/legal.html`의 원문으로 사용하는 생성 절차를 도입했다.
- 정렬 순위 표시를 화면에 보이는 상품만 대상으로 하고, 표시 항목 숨김 뒤 정렬을 다시 적용하도록 보완했다.
- 팝업에서 의견 보내기 진입을 제목 바로 옮기고 의견 화면 높이를 맞췄다.

## 2.0.3

- 확장 팝업과 소개 웹의 의견 시각에서 시·분을 제거하고 날짜만 표시했다.

## 2.0.2

- 소개 웹의 의견 작성 폼을 제거했다. 이후 의견 제출은 확장 팝업에서만 가능하고 웹은 목록 조회만 제공한다.
- 등록 직후 페이지 offset, `더 보기` 중복 클릭, 창 크기·폰트 로딩 뒤 3줄 말줄임 계산을 보완했다.
- 제한 안내를 `하루에 2개까지 보낼 수 있습니다`로 통일했다.

## 2.0.1

- 의견 본문을 기본 3줄로 접고 필요한 경우에만 `펼치기`/`접기`를 표시했다.
- 팝업·웹·API의 기본 페이지 크기를 10건에서 5건으로 변경했다.

## 2.0.0

호환성을 깨는 저장 키·브랜딩 전환이므로 메이저로 둔다.

- 사용자 표시명을 `알뜰이`, 기능명을 `페이지 표시 항목 조정`으로 통일하고 아이콘·팝업·법적 고지의 시각 체계를 정리했다.
- 새 설치의 모든 기능을 opt-in으로 바꿨다. 기존의 같은 이름 설정은 기본값 병합으로 유지했다.
- 내부 `cra`/`addon` 접두사를 `alt`로 변경하고 설정 스키마를 8로 올렸다. 구 접두사 키의 자동 변환은 제공하지 않았다.
- 의견 작성·최근 목록을 확장 팝업과 소개 웹에 추가했다. 이 당시 웹에도 작성 기능이 있었으나 2.0.2에서 제거됐다.
- 의견은 최대 500자, 동일 IP의 제출은 UTC 하루 2건으로 제한했다. 원문 IP 대신 날짜별 HMAC 식별값과 횟수를 저장하도록 했다.

## 1.2.1

- 실제 장바구니 완료 UI의 `담기 완료`, `상품이 장바구니에 담겼습니다`, 장바구니 이동 버튼을 성공 판정에 추가했다.

## 1.2.0

검색 결과에서 숨김 iframe으로 상품 상세를 열고 쿠팡 네이티브 담기 버튼을 실행하는 `장바구니 바로 담기`를 도입·안정화한 라인이다. (구 문서에서 1.15.0~1.15.6으로 나뉘어 있던 출시·중간 실험을 한 마이너로 정리했다.)

- 스피너·성공·실패 상태만 표시하고 수량 조절이나 별도 토스트는 두지 않았다.
- 당시 새 설정의 기본값은 켜짐이었으나 2.0.0부터 새 설치 기본값은 꺼짐이다.
- 정렬 순위·담기 버튼·법적 고지 스타일을 팝업 블루 중심으로 맞췄다.
- 카드 호버 시 iframe 최대 3개 프리패시는 곧 폐기하고, 담기 버튼 160ms 워밍·iframe 1개·요청 후 폐기로 축소했다.
- 쿠팡 CSP에 막히던 인라인 훅은 `extension/page-cart-hook.js` MAIN world 주입으로 바꿨다.
- 헤더 장바구니 숫자·타임아웃 성공 추정을 제거하고, 완료 UI·응답만 성공으로 인정했다. (완료 문구 보강은 1.2.1)

## 1.1.0

제작자 고정 프리셋·팝업 중심 UI·호스트 확대가 자리 잡은 라인이다. (구 1.7.0~1.14.0)

- 요소 피커·드래그 선택·캡처 썸네일·실행취소·사용자 다중 프리셋·`activeTab`을 제거하고 `extension/preset-data.js` 고정 프리셋으로 전환했다. (당시 체크 의미는 「보이기 예외」였고, 체크=표시는 2.1.0)
- 표시 항목 선택을 팝업으로 옮기고 `storage.onChanged`로 즉시 반영했다.
- 팝업 그룹형 목록·글래스/iOS 시각 실험을 했다가 흰 배경·팝업 블루로 되돌렸다.
- 표시 항목 상세 검색(초성·자모), 검색 개수 36·48·60·72 토글, 장바구니·주문목록 호스트(`cart`/`mc`)와 관련 프리셋을 추가했다.
- 가격 추출을 텍스트 패턴·단위 정규화로 바꾸고, 정렬 상태를 `chrome.storage.local`에 저장해 같은 검색어 리로드에서 복원했다.
- 개인정보처리방침·이용약관·최초 설치 법적 고지를 추가했다. (의견 서버는 아직 없음. 2.0.0 이후는 현행 방침)

## 1.0.2

페이지에서 요소를 직접 고르는 실험 구간이다. (구 1.3.0~1.6.x) 1.1.0에서 고정 프리셋으로 대체되며 폐기됐다.

- 하드코딩 광고 제거 → 요소 선택·드래그·썸네일·실행취소·다중 사용자 프리셋.
- 이후 제작자 배포 프리셋 하나로 단순화하고 항목을 검증·정리했다.

## 1.0.1

키워드 필터·설정 스키마 안정화다. (구 1.2.0~1.2.8)

- 키워드 UI·상품명 탐색 보강, 검색어 `q`/폼 재제출 기준 초기화, 제외 태그 마크업 정리.
- 업데이트 시 설정 덮어쓰기 방지, `settingsVersion` 병합 도입.
- 광고 표식 있는 경우로 숨김을 제한하고 브랜드 배너 숨김을 추가했다.

## 1.0.0

- 단위가·할인율·가격 정렬, 배너·프로모션 숨김, 키워드 필터, 검색 개수 고정을 포함한 첫 저장소 버전이다.
- 곧이어 검색 결과의 `Ad information`/`광고` 스폰서 상품 숨김을 추가했다.
