# QC — 출시 전에 뭐 보면 되나

예전에 고친 버그가 **다시 안 나오는지**, 기능이 **지금 문서대로 도는지** 확인하는 메모다.  
근거는 [`버전.md`](버전.md)·[`업그레이드.md`](업그레이드.md)·[`방법론.md`](방법론.md)다.

---

## 1. 컴퓨터가 대신 해주는 것 (먼저)

저장소 루트에서:

```bash
node --test tools/tests/extension/*.mjs
```

전부 통과하면 아래는 이미 코드 쪽으로 확인된 것이다.

- 설치만 하면 기능이 전부 꺼져 있음
- 「알뜰이 켜기/끄기」 같은 전역 스위치가 없음
- 숨긴 상품이 정렬 번호에 안 들어감
- 검색 개수 라디오를 눌러 페이지를 새로고침하지 않음
- 표시 항목은 「체크 해제한 것만 숨김」
- 소개 사이트에는 의견 **쓰기**가 없고 **목록만**
- 의견은 5개 단위, 날짜만 보임
- 장바구니 담기 성공은 「완료 문구·응답」으로만 판단 (헤더 숫자로 추측하지 않음)
- 예전에 쓰던 요소 피커·캡처 권한이 다시 안 들어옴
- 로드 순서에 `shared-start` · `page-runtime` · `settings-bridge`가 있음

**2.2.2 기준:** 2026-07-21에 위 테스트 **33개 전부 통과**.

## 2. 내가 손으로 보는 것 (쿠팡 + 팝업)

Chrome에서 `extension/` 폴더를 **Load unpacked** 한 뒤 한다.

### 팝업

1. 아이콘 클릭 → 기능이 전부 꺼져 있는지
2. 하나씩 켜고 끄기 → 다른 기능을 묶어서 끄지 않는지
3. 「페이지 표시 항목 조정」 세부 → 체크 해제한 항목만 숨기는지
4. 「의견 보내기」 → 안내 문구 있고, 보내면 목록에 나타나는지

### 쿠팡 검색 (`/np/search`)

| 볼 것 | 이렇게 되면 OK |
|--------|----------------|
| 정렬 | 단위가격·할인율·가격 버튼이 뜨고, 누르면 목록이 바뀜. 같은 검색어로 새로고침해도 정렬이 유지됨 |
| 키워드 필터 | 단어 넣으면 그 상품이 사라짐. 검색어를 바꾸면 제외 키워드가 비워짐 |
| 표시 항목 | 팝업에서 뺀 광고/배너가 안 보임. 기능 끄면 쿠팡 화면이 원래대로 |
| 검색 개수 | 켠 개수(예: 72)로 URL이 맞춰지고, 우리 버튼·필터가 사라지지 않음 |
| 장바구니 담기 | 상품 위에 담기 버튼 → 로그인·옵션이 필요하면 그걸 우회하지 않고, 성공/실패가 버튼 상태로 보임 |

### 소개 사이트

[https://altteuri.vercel.app](https://altteuri.vercel.app) → 「최근 의견」만 보이고, 글 쓰는 칸이 없으면 OK.

### 예전에 쓰던 설치에서 업데이트할 때

예전 버전에서 2.2.2으로 올리면:

- 켜 두었던 개별 기능·표시 항목은 그대로인지
- 저장소 `settingsVersion`이 4로 고쳐지는지 (구 10 등이었으면)
- 2.2.0 이전에서 오면 [`업그레이드.md`](업그레이드.md)의 마스터 스위치·체크 의미·저장 키 항목도 함께 본다

## 3. 한 줄 요약

1. `node --test tools/tests/extension/*.mjs` → 초록  
2. 팝업 + 쿠팡 검색에서 위 표 한 바퀴  
3. 소개 사이트에 의견 작성칸 없는지  

다 되면 이 버전 QC는 끝난 거다.

다음 버전을 낼 때는 이 파일의 버전·날짜·테스트 통과 기록만 갱신하면 된다.
