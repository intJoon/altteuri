# 방법론

## 코딩 docs 워크플로 (이 저장소)

- 코드에 주석 없음. 설명·결정은 `docs/방법론.md`, 외부 근거는 `docs/출처.md`, 폐기·대체는 `docs/업그레이드.md`, SemVer SSOT는 `docs/버전.md`.
- `docs/버전.md`와 `manifest.json` version 일치. 버전 항목 1개 = git 커밋 1개, 메시지 첫 줄 `X.Y.Z: <요약>`.

## 프리셋 데이터 (`preset-data.js`) 작성 규칙

- `window.ALT_BUILTIN_PRESET = { name, version, items:[{ selector, name, category }] }`.
- `category`: `srp` | `pdp` | `cart` | `order` | `etc`. 팝업 `ALT_GROUPS`와 대응.
- `name`은 원본 화면 제목과 맞추고, 없으면 적절히 명명. `items`는 페이지 위→아래 순서.
- 표시 조정 대상은 요소별 분리. 두 페이지 공통 요소는 페이지 고유 조상으로 좁힘.
- 제작자: CDP 등으로 실측 후 items에 직접 반영. 사용자: 팝업에서 `altPresetOff`로 개별 표시(기능 on일 때 기본은 숨김, 기능 자체는 설치 기본 off).

## 2026-07-17–19 — 스토어 보수화·브랜딩·의견 보내기 (v1.16.0)

- **릴리스 범위:** HEAD 1.15.6 이후 아직 커밋하지 않은 스토어 보수화, 브랜딩·내부 식별자 정리, 의견 웹/API와 요청 보호를 하나의 1.16.0 릴리스로 묶는다. 버전 항목 1개 = 커밋 1개 원칙에 따라 중간 작업 번호는 별도 릴리스로 취급하지 않는다.
- **기본값·표기:** 최초 설치에서 `DEFAULT_SETTINGS`의 모든 기능을 false로 두고 content/popup 읽기 폴백도 opt-in으로 통일. 기존 사용자 sync 값은 merge로 유지한다. 사용자 표시명은 `알뜰이`, 기능명은 `페이지 표시 항목 조정`으로 통일하며 영문 이름은 저장소·패키지·URL 식별자에만 남긴다.
- **내부 식별자:** 구 `cra`/`addon*` 접두를 `alt`로 교체한다. 저장 키는 `altEnabled`·`altPresetOff`·`altActiveSort`·`altSortQuery`, 프리셋 전역은 `ALT_BUILTIN_PRESET`, DOM·CSS·메시지·MAIN world 훅은 `alt`/`__alt*` 접두를 사용한다. `settingsVersion`은 8이며 구 키 호환 마이그레이션은 두지 않는다.
- **아이콘·화면:** '알뜰이' 초성 **ㅇㄸㅇ**를 얼굴로 재배치해 16px에서도 얼굴 하나로 읽히게 하고 팝업 블루 `#346aff` 바탕에 적용한다. `[KIM]WILDgag-Bold` 글리프를 `tools/make-icons.py`에서 슈퍼샘플링해 크기별 아이콘으로 만든다. `legal.html`도 같은 아이콘·팔레트를 사용하며, 팝업 상세 페이지는 메인 높이를 유지하고 목록만 스크롤한다.
- **의견 구조:** 확장 팝업과 소개 웹페이지에서 이용자가 선택적으로 의견(최대 500자)을 작성하고 최근 의견을 볼 수 있게 한다. Vercel 서버리스 API를 Neon의 유일한 진입점으로 두어 본문·자료형을 검증하고 자격증명을 서버에만 둔다. 소개 페이지는 동일 출처로, 확장은 manifest의 `https://altteuri.vercel.app/*` 권한으로 호출하며 API는 origin과 메서드를 제한한다.
- **제출 제한:** 원문 IP를 저장하지 않고 `HMAC-SHA256(RATE_LIMIT_SECRET, UTC 일자 + IP)`로 일별 해시를 생성한다. Neon의 `(day, ip_hash)` 레코드를 원자적으로 갱신하고 의견 INSERT와 한 SQL 문으로 묶어 동일 IP의 UTC 하루 2건만 등록한다. 원문 IP는 요청 중에만 사용하고 일별 해시·횟수는 의견과 분리해 보관하며, 7일이 지난 레코드는 후속 POST에서 삭제한다.
- **개인정보·콘텐츠 안전:** 1.13.0 당시의 “무수집·외부 서버 없음” 전제는 선택 제출 기능 도입으로 대체한다. 일반 기능의 브라우징 기록·계정·구매 내역은 수집하지 않고 설정값도 브라우저에만 남지만, 제출한 의견·확장 버전과 요청 보호 정보는 Vercel·Neon에서 처리한다. 의견은 일반 문자열로 저장하고 `textContent`로 렌더링한다.

## 2026-07-12 — 커스텀 정렬 ↔ 팝업 설정 상호 영향 제거 (v1.12.2)

- **문제:** 커스텀 정렬을 켠 상태에서 팝업 설정을 건드리면 리로드되며 정렬이 풀림. "이런 상호 영향의 경우의 수" 전수 조사 요청.
- **조사 결과(상호 영향 경우의 수):**
  - (A 근본원인) `popup.js`의 8개 핸들러가 예외 없이 `chrome.tabs.reload()` 호출. 커스텀 정렬 활성 플래그(`isUnitPriceSortActive`/`isDiscountRateSortActive`/`isPriceSortActive`)는 메모리 변수이고 storage에 없어서 리로드 시 소실. 복원 함수도 메모리 플래그 의존이라 리로드 후 복원 불가.
  - (B 중복) content는 키워드 필터·광고요소 제거를 `storage.onChanged`로 이미 즉시 반영하는데 팝업이 추가로 리로드까지 함 → 불필요 리로드가 정렬을 죽이는 부수효과. 정렬 3종·검색 개수는 반대로 content에 리스너가 없어 리로드 의존(비대칭).
  - (C) 검색 개수 변경은 `listSize` URL 파라미터를 바꾸는 네비게이션(`location.replace`)이라 본질적으로 페이지가 새로 뜸.
  - (D) 정렬 버튼으로 끌 때 원래 순서 복구용 `location.reload()`(의도적). 팝업 리로드는 활성 탭뿐 아니라 모든 `www.coupang.com` 탭 대상.
  - (E) 팝업 리로드는 `www.coupang.com`만 → `cart.coupang.com` 미대상(정렬은 검색페이지 전용이라 영향 적음).
- **결정(각 문제를 나은 방향으로):**
  1. 활성 정렬 상태를 `chrome.storage.local`에 영속화(`craActiveSort`=unit|discount|price|null, `craSortQuery`=대상 q). 검색어를 함께 저장해 **같은 검색어 리로드에서만** 복원 → 새 검색엔 리로드 경쟁 없이 안전하게 복원 안 함(레이스 회피, 기존 "새 검색=초기화" UX 유지).
  2. 복원 함수(`restore*Sort`)를 메모리 플래그가 아니라 `getActiveSort()`(storage) 기준으로 재작성. `sortBy*` 진입 시 `saveActiveSort(kind)`, 끄기/새검색/마스터오프 시 `saveActiveSort(null)`.
  3. 팝업의 정렬·광고제거·키워드·검색 개수 핸들러에서 `tabs.reload` 제거. content에 `initSortAndSizeSync()`(onChanged) 추가: 정렬 토글=버튼 주입/제거(`handleSortFeatureToggle`), 검색 개수=해당 탭에서 `setListSizeFromAddon()`. 마스터 토글만 리로드 유지(전역 on/off라 리로드가 자연스럽고, 라이브 부트스트랩은 위험/복잡).
  4. 정렬 기능을 끌 때 그 정렬이 활성 중이면 원래 순서 복구 위해 리로드(단, `saveActiveSort(null)` 선반영으로 재적용 방지). 비활성 정렬 끄기는 리로드 없음.
- **부수 효과 개선:** 검색 개수 변경 후에도 정렬 복원됨(네비게이션 후 같은 검색어면 재적용). 백그라운드 탭 불필요 리로드 감소.
- **핫픽스(선언 순서 TDZ):** 위 커밋 직후 "설정 다 켜져있는데 적용이 안 됨" 제보. 원인은 최상단 `initSortAndSizeSync()`(동기 호출)가 참조하는 `let sortSizeChangeBound`를 호출부보다 아래에 선언한 것 — `let`은 함수 선언과 달리 호이스팅되지 않아 TDZ `ReferenceError`가 나고 그 뒤 `isExtensionEnabled(...)` 초기화 블록 전체가 실행 중단됨(`node --check`는 구문만 봐서 못 잡음). `let sortSizeChangeBound = false;`를 정렬 플래그 옆(상단)으로 이동해 해결. (참고: `kwChangeBound`는 async storage 콜백 안에서만 접근돼 실제 크롬에선 안전 — 동기 mock에서만 걸림.)
- **검증 방법(중요):** 이 Chrome 빌드에서 `--load-extension`이 동작하지 않아(공백 경로 문제 + 로드 제약) 확장 자체 로드 검증이 불가. 대신 mock `chrome`(runtime.id + 비동기 storage.sync/local + onChanged)을 페이지에 주입하고 `preset-data.js`+`content.js`를 실제 쿠팡 페이지에서 실행해 최상단 예외 없음 + 정렬 3종 버튼/키워드 필터/스타일 주입 적용을 확인. storage 콜백을 반드시 비동기(`Promise.resolve().then`)로 mock해야 실제 환경과 동일(동기 mock은 top-level TDZ 오탐 유발).
- **검증:** `node --check` content/popup 통과, ReadLints 무오류, mock 주입 실행 검증 통과.

## 2026-07-11 — 프리셋 + 요소 분류·재명명·표시 토글 + 팝업 분류 (v1.5.0)

- **데이터 모델:**
  - 작업 셋 = `hiddenElementSelectors`(sync) + `craHiddenMeta`(local). 메타 항목에 `name`(재명명)·`category`·`enabled`(표시 여부) 확장. `label`은 자동 추출 텍스트, `name`은 사용자 지정(우선 표시).

  - 적용 = 프리셋 selectors를 작업 셋으로 교체 + items를 메타에 병합(기존 썸네일 보존) + 재적용.

- **팝업:** 섹션 카드(정렬 / 광고·요소 제거+프리셋 / 키워드 / 검색 표시)로 재구성. 프리셋 관리(select + 적용/이름변경/삭제 + 현재 상태 저장)는 sync `craPresets` + local `craHiddenMeta`를 직접 읽고 적용 후 쿠팡 탭 reload로 반영.
- **결정:** 썸네일은 프리셋에 저장하지 않음(용량) — 적용 시 기존 메타에 있으면 재사용, 없으면 플레이스홀더 후 재선택 시 재캡처. 요소 상세 조정(분류/재명명/표시)은 DOM 접근이 필요해 페이지 툴에 두고, 팝업은 프리셋 단위 관리로 역할 분담.

## 2026-07-11 — 확장 프로그램 재로드 시 설정 초기화

- **문제:** `chrome://extensions`에서 업데이트·재로드할 때마다 popup 설정이 기본값으로 돌아감.
- **원인:** `background.js`의 `onInstalled`가 install/update/개발자 재로드마다 `storage.sync.set`으로 전체 기본값 덮어쓰기.
- **선택:** `addonEnabled`가 이미 storage에 있으면 기본값 주입 생략. 최초 설치(`undefined`)일 때만 `DEFAULT_SETTINGS` 적용.
- **한계:** 설정 키가 늘어나도 기존 사용자에게 새 키 기본값이 안 들어감 → v1.2.3에서 `settingsVersion` + `mergeWithDefaults()` 마이그레이션으로 교체.

## 2026-07-11 — settingsVersion 마이그레이션

- **문제:** `addonEnabled` 존재 여부만 보면 유지보수 시 (1) 새 설정 키 추가 시 기존 사용자 누락, (2) sentinel 키 변경 시 오동작.
- **선택:**
  - `SETTINGS_VERSION` 상수 + storage `settingsVersion` 필드.
  - 최초 설치: `DEFAULT_SETTINGS` 전체 저장.
  - 재로드·업데이트·구버전(`settingsVersion` 없음 + `addonEnabled` 있음): `mergeWithDefaults()` — 기본값 위에 기존 값 덮어써 사용자 설정 유지, 새 키만 기본값 채움.
  - `SETTINGS_VERSION` 올릴 때마다 `DEFAULT_SETTINGS`에 새 키 추가 후 버전 bump.
