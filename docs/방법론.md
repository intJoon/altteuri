# 방법론

이 문서는 알뜰이 1.16.4의 현재 구조와 검증 원칙을 설명한다. 릴리스별 변화는 [`버전.md`](버전.md), 폐기·호환성 이력은 [`업그레이드.md`](업그레이드.md), 외부 근거와 라이선스는 [`출처.md`](출처.md)를 따른다.

## 제품 원칙

- 알뜰이는 쿠팡과 제휴하지 않은 비공식 보조 도구다. 쿠팡의 원래 구매 흐름과 정책을 대체하지 않는다.
- 최초 설치 시 `background.js`의 모든 기능 기본값은 `false`다. 사용자가 팝업에서 기능을 켜야 동작한다.
- 일반 기능은 브라우징 기록, 계정 정보, 구매 내역을 개발자 서버로 보내지 않는다.
- 의견 목록 조회와 의견 제출만 `https://altteuri.vercel.app`의 API를 사용한다. 소개 웹은 목록 조회 전용이며 제출은 확장 팝업에서만 가능하다.

## 실행 구조

### 확장 프로그램

- `manifest.json`: Manifest V3 권한, 대상 호스트, 콘텐츠 스크립트와 서비스 워커를 선언한다.
- `background.js`: 최초 설치 기본값 병합, 최초 설치 시 법적 고지 열기, 장바구니 기능의 MAIN world 스크립트 주입을 담당한다.
- `popup.html` / `popup.js`: 설정, 표시 항목 선택, 의견 작성·조회 UI를 제공한다.
- `preset-data.js`: 제작자가 검증한 표시 조정 대상과 페이지 분류를 제공한다.
- `content.js`: 쿠팡 페이지에서 정렬, 키워드 필터, 검색 개수 고정, 표시 항목 조정, 장바구니 바로 담기를 적용한다.
- `page-cart-hook.js`: 숨김 상품 상세 iframe의 MAIN world에서 장바구니 관련 fetch/XHR 결과와 클릭을 관찰한다.
- `hangul-search.js`: 팝업의 프리셋 이름을 완성형·자모·초성으로 검색한다.

콘텐츠 스크립트는 `www.coupang.com`, `cart.coupang.com`, `mc.coupang.com`에서 실행된다. 기능별 경로 가드를 함께 사용하므로 검색 전용 기능은 `/np/search` 밖에서 적용하지 않는다.

### 의견 서비스

- 확장 팝업은 의견 본문과 확장 버전을 `POST /api/comments`로 보낸다.
- 확장 팝업과 소개 웹은 `GET /api/comments`로 최근 의견을 5개씩 조회한다.
- `web/api/comments.mjs`가 입력 길이·자료형·허니팟·페이지 범위·메서드를 검증하고, `web/lib/db.mjs`를 통해 Neon PostgreSQL에 접근한다.
- 의견 본문은 DOM에 `textContent`로 넣어 HTML로 해석하지 않는다.
- API 자격증명과 요청 제한 비밀키는 서버 환경 변수에만 둔다.

## 설정과 데이터 흐름

### 브라우저 저장소

`chrome.storage.sync`에는 다음 사용자 설정이 저장된다.

- 마스터와 기능별 on/off
- 정렬 방향, 검색 개수, 제외 키워드와 해당 검색어
- 표시하지 않을 프리셋의 예외 목록 `altPresetOff`
- 마스터를 다시 켤 때 복원할 마지막 기능 조합

`chrome.storage.local`에는 현재 검색어에 적용한 정렬 상태(`altActiveSort`, `altSortQuery`)와 전송 전 의견 초안(`altFeedbackDraft`)을 저장한다. 설정은 브라우저 저장소에만 있으며, sync 데이터의 브라우저 계정 동기화 여부는 브라우저 정책에 따른다.

현재 설정 스키마는 `settingsVersion: 8`이다. 업데이트 시 `mergeWithDefaults()`가 기존 키를 보존하면서 새 기본 키만 채운다. 1.16.0 이전의 `cra`/`addon` 키는 자동 변환하지 않는다.

### 표시 항목 프리셋

`preset-data.js`의 형식은 다음과 같다.

```js
window.ALT_BUILTIN_PRESET = {
  name,
  version,
  items: [{ selector, name, category }]
};
```

`category`는 `srp`, `pdp`, `cart`, `order`, `etc` 중 하나다. 프리셋은 제작자가 배포하는 고정 목록이며 사용자가 임의 셀렉터를 만드는 기능은 없다.

표시 항목 조정 기능을 켜면 프리셋 항목은 기본적으로 숨겨지고, 사용자가 체크를 해제한 셀렉터만 `altPresetOff`에 저장되어 보인다. 기능 자체가 최초 설치 시 꺼져 있으므로 설치 직후 페이지를 변경하지 않는다.

## 기능별 처리 원칙

### 정렬

- 상품 목록과 상품명은 의미 있는 클래스 접두·DOM 구조로 찾고, 가격·할인율·단위가는 텍스트 패턴을 주 기준으로 읽는다.
- 단위가는 `금액 / 기준량`으로 정규화해 기준량이 다른 상품을 비교한다.
- 적용한 정렬 종류와 검색어를 local 저장소에 기록해 같은 검색어의 재로딩에서만 복원한다.
- 상품을 읽을 수 없는 항목은 억지로 값을 추정하지 않고 정렬 가능한 항목 뒤에 둔다.

### 키워드 필터

- 제외 키워드는 상품명에 대한 대소문자 무시 부분일치로 적용한다.
- 검색어 `q`가 바뀌거나 검색 폼을 다시 제출하면 제외 키워드를 초기화한다.
- 쿠팡 자체 필터·정렬·페이지·목록 개수 변경만으로는 초기화하지 않는다.
- 쿠팡의 부분 렌더링 뒤에도 반복 재적용하되, 기능을 끄면 주입 UI와 숨김 상태를 제거한다.

### 검색 개수와 표시 항목

- 검색 개수 고정은 `/np/search`에서만 URL의 `listSize`를 36·48·60·72 중 선택값으로 맞춘다.
- 표시 항목은 `.alt-force-hidden` 클래스로 숨기며 DOM 변경 후 다시 적용한다.
- 자체 UI는 `data-alt-ui` 등의 표식으로 숨김 대상에서 제외한다.

### 장바구니 바로 담기

- 검색 결과의 상품 식별자를 이용해 같은 출처의 상품 상세를 숨김 iframe으로 연다.
- 담기 버튼 진입 시 160ms 뒤 iframe 하나만 미리 준비하고, 누르면 쿠팡의 네이티브 장바구니 버튼을 실행한다.
- 성공은 장바구니 요청 응답, 확인된 완료 문구, 장바구니 이동 버튼으로만 판정한다.
- 헤더 장바구니 숫자 변화나 타임아웃을 성공으로 추정하지 않는다. 시간 초과는 실패로 처리한다.
- 로그인, 옵션 선택, 멤버십, 구매 수량 제한 등 쿠팡이 요구하는 조건을 우회하지 않는다.

## 의견 처리와 보존

- 본문은 공백 제거 후 1~500자만 허용한다. 확장 버전은 선택 문자열이며 최대 50자다.
- 동일 IP의 제출은 UTC 날짜 기준 하루 2건이다.
- 원문 IP는 요청 처리 중에만 사용한다. `HMAC-SHA256(RATE_LIMIT_SECRET, UTC 날짜 + IP)`로 일별 식별값을 만들고 날짜·해시·횟수만 별도 테이블에 저장한다.
- 제한 갱신과 의견 삽입은 하나의 SQL 문에서 처리해 동시 요청에도 제한을 유지한다.
- 의견과 버전은 최대 730일 보관하며 GET 또는 POST 요청 때 만료분을 삭제한다.
- 요청 제한 식별값은 7일 보관 정책이며 후속 POST 때 오래된 레코드를 삭제한다.
- 의견은 공개 목록에 표시될 수 있으므로 제출 전 미국 소재 서비스 저장·공개 가능성과 개인정보 입력 금지를 알린다.

정확한 개인정보 처리 내용은 [`개인정보처리방침.md`](개인정보처리방침.md)를 우선한다.

## 검증 원칙

1. `manifest.json` 버전과 [`버전.md`](버전.md)의 최신 버전을 맞춘다.
2. JavaScript 변경은 최소한 `node --check`로 구문을 확인한다.
3. 설정 기본값은 `background.js`, 팝업의 누락값 처리, 콘텐츠 스크립트의 활성 조건을 함께 확인한다.
4. 쿠팡 DOM에 의존하는 셀렉터는 실제 페이지에서 존재 범위·오탐·위치를 확인하고, 관찰 날짜와 근거를 [`출처.md`](출처.md)에 남긴다.
5. 쿠팡 마크업이 없거나 해석이 불확실하면 성공·광고·가격을 추정하지 않는다.
6. 의견 API는 GET/POST, 입력 경계, 일일 제한, 동시성, 보존 삭제, CORS, HTML 비해석을 함께 검증한다.
7. 문서 검수 시 현재 동작과 당시 릴리스 사실을 분리하고, 폐기된 식별자나 실험은 현행 설명에 사용하지 않는다.
