# 방법론

이 문서는 알뜰이 2.2.3의 현재 구조와 검증 원칙을 설명한다. 릴리스별 변화는 [`버전.md`](버전.md), 폐기·호환성 이력은 [`업그레이드.md`](업그레이드.md), 외부 근거와 라이선스는 [`출처.md`](출처.md)를 따른다.

## 제품 원칙

- 알뜰이는 쿠팡과 제휴하지 않은 비공식 보조 도구다. 쿠팡의 원래 구매 흐름과 정책을 대체하지 않는다.
- 최초 설치 시 모든 기능 기본값은 `false`다. 사용자가 팝업에서 기능을 켜야 동작한다.
- 일반 기능은 브라우징 기록, 계정 정보, 구매 내역을 개발자 서버로 보내지 않는다.
- 의견 목록 조회와 의견 제출만 `https://altteuri.vercel.app`의 API를 사용한다. 소개 웹은 목록 조회 전용이며 제출은 확장 팝업에서만 가능하다.

## 실행 구조

- `extension/` — Manifest V3 확장. 설정 기본값(`settings-defaults.js`), 서비스 워커, 팝업, 콘텐츠 스크립트, 고정 프리셋, DOM 비의존 순수 로직.
- `web/` — 소개 사이트와 의견 API(Vercel + Neon).
- `docs/` — 방법론·버전·업그레이드·출처·법적 고지·QC
- `tools/` — 법적 고지 생성, 아이콘, 검사 스크립트, 확장 테스트(`tools/tests/`)

콘텐츠 스크립트는 `www.coupang.com`, `cart.coupang.com`, `mc.coupang.com`에서 실행되며, 검색 전용 기능은 `/np/search`에만 적용한다. `document_start`는 `preset-data` → `shared-start` → `early`, `document_idle`은 `preset-data` → `pure-logic` → `shared-start` → `core` → `keyword-filter` → `sort` → `list-size` → `quick-cart` → `element-remover` → `page-runtime` → `settings-bridge` → `boot` 순으로 로드한다. 같은 isolated world에서 `globalThis.Altteuri`와 `AltteuriShared`를 공유한다. `page-cart-hook.js`는 서비스 워커가 MAIN world에 별도 주입한다.

`shared-start`는 early/idle이 같이 쓰는 검색 개수 리다이렉트·표시 숨김 CSS 생성기다. `settings-bridge`는 `chrome.storage.onChanged`를 한곳에서 받아 기능 모듈로 넘긴다. `page-runtime`은 DOM·URL 변경 시 기능 재적용을 담당하고, `sort`는 정렬 UI·정렬 실행만 담당한다.

## 설정과 데이터 흐름

- `chrome.storage.sync`: 기능 on/off, 정렬 방향, 검색 개수, 제외 키워드, 표시하지 않도록 해제한 프리셋 목록.
- `chrome.storage.local`: 현재 검색어의 정렬 상태, 전송 전 의견 초안.
- 설정 스키마는 `settingsVersion: 10`이며 단일 소스는 `extension/settings-defaults.js`다. 업데이트 시 기존 키는 보존하고 새 기본 키만 채운다. 키 삭제·변환 이력은 [`업그레이드.md`](업그레이드.md)를 따른다.

표시 항목은 `extension/preset-data.js`의 제작자 고정 프리셋만 사용한다. 기능을 켜면 항목은 기본 표시이고, 체크를 해제한 항목만 숨긴다.

## 기능별 처리 원칙

### 정렬

- 가격·할인율·단위가는 텍스트 패턴을 주 기준으로 읽고, 단위가는 기준량으로 정규화한다.
- 읽을 수 없는 항목은 추정하지 않고 정렬 가능 항목 뒤에 둔다.
- 같은 검색어에서만 정렬 상태를 복원하며, 숨긴 상품은 순위 배지에 넣지 않는다.

### 키워드 필터

- 상품명 대소문자 무시 부분일치로 제외한다.
- 검색어 `q` 변경 또는 검색 폼 재제출 때 제외 키워드를 초기화한다.

### 검색 개수와 표시 항목

- `/np/search`에서 URL `listSize`를 선택값(36·48·60·72)으로 맞춘다. 라디오는 시각만 맞추고 `change`를 보내지 않는다. `document_start`와 idle 경로가 겹쳐도 같은 대상은 한 번만 `replace`한다.
- 프리셋 셀렉터에 CSS로 `display:none`을 주입하며, 자체 UI는 숨김 대상에서 제외한다. 숨긴 상품은 정렬 순번에서 뺀다.
- 커스텀 UI는 짧은 reconcile(약 30ms + MutationObserver)로 빠진 것만 보강한다. 쿠팡 SPA DOM 교체 시 잠깐 사라졌다가 다시 붙을 수 있다.

### 장바구니 바로 담기

- 숨김 iframe으로 상품 상세를 열고 쿠팡 네이티브 담기 버튼을 실행한다.
- 성공은 요청 응답과 확인된 완료 UI로만 판정한다. 쿠팡이 요구하는 로그인·옵션·멤버십·수량 제한을 우회하지 않는다.

## 의견

팝업에서 의견을 제출하고, 확장·소개 웹에서 최근 공개 목록을 조회한다. 보존·제한·위탁 세부는 [`개인정보처리방침.md`](개인정보처리방침.md)를 우선한다.

## 검증 원칙

1. `extension/manifest.json` 버전, [`버전.md`](버전.md)의 최신 버전, `settingsVersion`을 맞춘다.
2. JavaScript 변경은 최소한 `node --check`로 구문을 확인한다. 계약·순수 로직은 `node --test tools/tests/extension/*.mjs`로 확인한다.
3. 설정 기본값, 팝업의 누락값 처리, 콘텐츠 스크립트의 활성 조건을 함께 확인한다.
4. 쿠팡 DOM 셀렉터는 실제 페이지에서 확인하고, 관찰 날짜와 근거를 [`출처.md`](출처.md)에 남긴다.
5. 쿠팡 마크업이 없거나 해석이 불확실하면 성공·광고·가격을 추정하지 않는다.
6. 문서 검수 시 현재 동작과 당시 릴리스 사실을 분리한다.
