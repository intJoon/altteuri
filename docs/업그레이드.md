# 업그레이드

이 문서는 실제 사용자 설정 호환성에 영향을 주는 변경과 현재 남아 있는 레거시 처리만 설명한다. 전체 릴리스 이력은 [`버전.md`](버전.md)를 참고한다.

## 현재 설치와 업데이트

- 현재 버전은 1.16.5, 설정 스키마는 `settingsVersion: 9`이다.
- 새 설치의 모든 기능은 꺼져 있다. 설치만으로 쿠팡 페이지의 표시나 동작을 바꾸지 않는다.
- 업데이트 시 `background.js`가 현재 기본값과 저장된 값을 병합한다. 이름이 같은 기존 설정값은 유지되고 새 키만 기본값으로 추가된다.
- 확장을 제거하면 Chrome이 확장 저장소를 삭제한다. 브라우저 동기화 데이터의 제거·복원 시점은 Chrome 계정 정책에 따를 수 있다.

## 1.16.5: 표시 항목 체크 의미 반전

1.16.5에서 페이지 표시 항목 조정의 체크 의미를 바꿨다.

- 이전: 기능을 켜면 프리셋은 기본 숨김이고, 체크 해제한 항목만 `altPresetOff`에 넣어 보이게 했다.
- 현재: 기능을 켜면 프리셋은 기본 표시이고, 체크 해제한 항목만 `altPresetOff`에 넣어 숨긴다. 기능이 꺼져 있으면 페이지를 바꾸지 않는다.

스키마를 9로 올리며, 기능이 켜져 있던 설치는 업데이트 시 보이는 항목 집합이 같도록 `altPresetOff`를 반전 변환한다. 기능이 꺼져 있던 설치는 목록을 비워 새 기본(켜면 모두 표시)에 맞춘다.

## 1.16.0: 저장 키 이름 변경

1.16.0에서 제품 내부 접두사를 `cra`/`addon`에서 `alt`로 바꿨다.

- `addonEnabled` → `altEnabled`
- `craPresetOff` → `altPresetOff`
- `craActiveSort` / `craSortQuery` → `altActiveSort` / `altSortQuery`
- 프리셋 전역, DOM 속성, CSS 클래스, 메시지 이름도 `ALT_*`, `alt-*`, `__alt*` 계열로 변경

구 키를 현행 키로 자동 복사하는 마이그레이션은 없다. 1.15.x 이하에서 바로 업데이트하면 기능 설정을 팝업에서 다시 선택해야 할 수 있다. 구 키가 저장소에 남아도 현행 코드가 기능 설정으로 읽지 않는다.

## 1.16.0: 기본값 변경

1.15.x까지 일부 기능은 새 설치에서 켜질 수 있었지만, 1.16.0부터 마스터·정렬·키워드 필터·페이지 표시 항목 조정·검색 개수 고정·장바구니 바로 담기가 모두 opt-in이다.

이 변경은 기존의 같은 이름 설정을 강제로 끄지 않는다. 다만 위 저장 키 개명으로 인해 구 접두사에만 값이 있는 설치는 현행 설정으로 인식되지 않는다.

## 1.16.0~1.16.4: 의견 기능

- 1.16.0에서 확장과 소개 웹의 의견 작성·조회가 추가됐다.
- 1.16.1에서 목록 기본 단위를 5건, 본문 표시를 기본 3줄로 변경했다.
- 1.16.2에서 소개 웹의 작성 기능을 제거했다. 현재 제출은 확장 팝업에서만 가능하고 웹은 목록 조회만 제공한다.
- 1.16.3에서 목록 시각을 날짜만 표시하도록 바꿨다.
- 1.16.4에서 제출 전 저장·공개·미국 처리 안내와 보존기간을 명확히 하고, 의견 API가 730일 경과 의견을 삭제하도록 했다.

의견 초안은 `chrome.storage.local`의 `altFeedbackDraft`에 임시 저장되며 전송 성공 시 삭제된다. 의견 본문과 확장 버전은 제출한 경우에만 서버에 저장된다.

## 1.15.3~1.15.6: 장바구니 바로 담기 안정화

- 1.15.2의 상품 카드 전체 호버 프리패치와 iframe 3개 캐시는 1.15.3에서 폐기됐다.
- 현재는 담기 버튼 위 160ms 워밍, iframe 한 개, 요청별 폐기를 사용한다.
- CSP에 막히던 인라인 주입은 외부 파일 `page-cart-hook.js`를 MAIN world에 주입하는 방식으로 교체됐다.
- 헤더 장바구니 숫자와 타임아웃 성공 추정은 제거됐다. 현재는 요청 응답과 확인된 완료 UI만 성공으로 인정한다.

이 기능은 쿠팡의 로그인·옵션·멤버십·수량 제한을 우회하지 않으며 페이지 변경에 따라 실패할 수 있다.

## 1.12.2: 정렬 상태 복원

정렬 상태를 메모리에만 두던 방식은 같은 검색어에서 리로드 후 복원할 수 있도록 `chrome.storage.local` 저장으로 교체됐다. 현행 키는 `altActiveSort`와 `altSortQuery`다.

정렬 외 설정 변경은 대부분 `chrome.storage.onChanged`로 반영한다. 마스터를 끄고 켤 때는 `www.coupang.com` 탭을 다시 로드한다.

## 1.7.0: 표시 항목 모델 교체

사용자가 페이지에서 요소를 직접 선택하던 다음 기능은 제거됐다.

- 사용자 셀렉터와 메타데이터
- 다중 사용자 프리셋
- 플로팅 피커, 드래그 선택, 캡처 썸네일, 실행취소·다시실행
- `activeTab`과 `captureVisibleTab`

현재는 `preset-data.js`의 제작자 고정 프리셋과, 표시하지 않도록 해제한 항목 목록 `altPresetOff`만 사용한다. 구 데이터는 읽거나 변환하지 않는다.

## 키워드 필터 레거시 읽기

현재 검색어 식별 키는 `excludedKeywordsForQuery`다. 과거 `excludedKeywordsSessionKey` 값은 기존 이용자의 검색어를 한 번 해석하기 위한 읽기 폴백으로만 남아 있다. 새 저장은 `excludedKeywordsForQuery`로만 한다.

제외 키워드는 검색어 `q` 변경 또는 검색 폼 재제출 때 초기화한다. 페이지·정렬·검색 개수·쿠팡 필터만 바뀌면 유지한다.

## 호환성 확인 체크리스트

1. `manifest.json`과 [`버전.md`](버전.md)의 버전이 같은지 확인한다.
2. `background.js`의 `SETTINGS_VERSION`과 `DEFAULT_SETTINGS.settingsVersion`이 같은지 확인한다.
3. 설정 키를 추가하면 기본값 병합과 팝업·콘텐츠 스크립트의 누락값 처리를 함께 확인한다.
4. 저장 키를 개명해야 한다면 유지할 사용자 값을 명시적으로 변환하거나, 변환하지 않는다는 사실을 릴리스 전에 고지한다.
5. 권한·호스트가 늘어나면 기능 범위와 개인정보처리방침을 함께 검토한다.
