# 업그레이드

이 문서는 실제 사용자 설정 호환성에 영향을 주는 변경과 현재 남아 있는 레거시 처리만 설명한다. 전체 릴리스 이력은 [`버전.md`](버전.md)를 참고한다. 구 문서의 `1.16.x` 등은 [`버전.md`](버전.md) 하단 대응표를 본다.

## 현재 설치와 업데이트

- 현재 버전은 2.2.4, 설정 스키마는 `settingsVersion: 10`이다.
- 새 설치의 모든 기능은 꺼져 있다. 설치만으로 쿠팡 페이지의 표시나 동작을 바꾸지 않는다.
- 업데이트 시 `extension/background.js`가 현재 기본값과 저장된 값을 병합한다. 이름이 같은 기존 설정값은 유지되고 새 키만 기본값으로 추가된다.
- 확장을 제거하면 Chrome이 확장 저장소를 삭제한다. 브라우저 동기화 데이터의 제거·복원 시점은 Chrome 계정 정책에 따를 수 있다.
- Load unpacked로 `1.16.x` / `2.2.1` / `2.2.2` / `2.2.3`를 쓰던 설치는 폴더를 다시 로드하면 매니페스트가 2.2.4로 올라간다. 스키마는 10이며, 2.2.2에서만 쓰인 연속 라벨 `3`/`4`는 10으로 정규화하되 표시 항목 목록은 다시 뒤집지 않는다.

## 2.2.4: 회색 아이콘·커스텀 정렬 순번 UI

설정 스키마·저장 키 마이그레이션은 없다.

- 기능 토글이 모두 꺼져 있으면 확장 아이콘이 회색(`icon*-gray.png`)으로 바뀐다.
- 커스텀 정렬 중에는 순번 뱃지만 표시하고, 기준 값 텍스트 배지는 붙이지 않는다.
- 커스텀 정렬 중 쿠팡 네이티브 순위는 CSS로만 숨기며, 정렬을 끄면 쿠팡 순번이 다시 보인다.
- 정렬 해제·기능 off·검색어 변경 시 커스텀 순번이 DOM에서 제거된다.

## 2.2.3: 문서 일관성·2.2.2 철회 경로

기능 동작은 2.2.1과 같다. 스키마 연속화(2.2.2)는 폐기했고, 약관·브랜드 표기를 제품 명칭에 맞췄다. 저장소에 `settingsVersion` 3 또는 4가 있으면 10으로 올리고 `altPresetOff` 반전은 건너뛴다.

## 2.2.2: 스키마 연속화 (철회)

저장 라벨을 1–4로 바꾸려다 철회했다. 새 설치·현행 코드는 스키마 **10**만 쓴다.

## 2.2.1: 콘텐츠 스크립트 구조·이력 재넘버링

사용자 설정 키·스키마를 바꾸지 않는다. 공통 early/idle 로직·페이지 재적용·설정 반영 경로를 파일로 나누고 회귀 테스트 위치를 `tools/tests/extension/`으로 옮겼다. 같은 배포에서 문서 SemVer만 구 `1.16.7` → `2.2.1` 체계로 정리했다.

## 2.2.0: 마스터 스위치 제거

팝업의 전역 「알뜰이 켜기/끄기」와 `altEnabled`·`lastPreset` 복원 경로를 제거했다.

- 기능은 각자 토글만으로 동작한다.
- 스키마를 10으로 올리며, 업데이트 시 저장소에서 `altEnabled`와 `lastPreset`을 지운다.
- 예전에 마스터만 끄고 기능 플래그를 모두 `false`로 둔 설치는 그대로다. 필요한 기능은 팝업에서 다시 켠다.

## 2.1.0: 표시 항목 체크 의미 반전

페이지 표시 항목 조정의 체크 의미를 바꿨다.

- 이전: 기능을 켜면 프리셋은 기본 숨김이고, 체크 해제한 항목만 `altPresetOff`에 넣어 보이게 했다.
- 현재: 기능을 켜면 프리셋은 기본 표시이고, 체크 해제한 항목만 `altPresetOff`에 넣어 숨긴다. 기능이 꺼져 있으면 페이지를 바꾸지 않는다.

스키마를 9로 올리며, 기능이 켜져 있던 설치는 업데이트 시 보이는 항목 집합이 같도록 `altPresetOff`를 반전 변환한다. 기능이 꺼져 있던 설치는 목록을 비워 새 기본(켜면 모두 표시)에 맞춘다. 변환 후에는 `www`·`cart`·`mc` 쿠팡 탭을 다시 불러 열린 페이지에도 새 의미를 적용한다.

## 2.0.0: 저장 키 이름 변경

제품 내부 접두사를 `cra`/`addon`에서 `alt`로 바꿨다. 자동 마이그레이션이 없어 메이저로 둔다.

- 당시 `addonEnabled` → `altEnabled` (2.2.0에서 `altEnabled` 자체 제거)
- `craPresetOff` → `altPresetOff`
- `craActiveSort` / `craSortQuery` → `altActiveSort` / `altSortQuery`
- 프리셋 전역, DOM 속성, CSS 클래스, 메시지 이름도 `ALT_*`, `alt-*`, `__alt*` 계열로 변경

1.x에서 바로 올리면 기능 설정을 팝업에서 다시 선택해야 할 수 있다.

## 2.0.0: 기본값 변경

1.2.x까지 일부 기능은 새 설치에서 켜질 수 있었지만, 2.0.0부터 정렬·키워드 필터·페이지 표시 항목 조정·검색 개수 고정·장바구니 바로 담기가 모두 opt-in이다. 같은 이름 설정을 강제로 끄지는 않는다.

## 2.0.0~2.0.4: 의견 기능

- 2.0.0에서 확장과 소개 웹의 의견 작성·조회가 추가됐다.
- 2.0.1에서 목록 기본 단위를 5건, 본문 표시를 기본 3줄로 변경했다.
- 2.0.2에서 소개 웹의 작성 기능을 제거했다. 현재 제출은 확장 팝업에서만 가능하고 웹은 목록 조회만 제공한다.
- 2.0.3에서 목록 시각을 날짜만 표시하도록 바꿨다.
- 2.0.4에서 제출 전 저장·공개·미국 처리 안내와 보존기간을 명확히 하고, 의견 API가 730일 경과 의견을 삭제하도록 했다.

의견 초안은 `chrome.storage.local`의 `altFeedbackDraft`에 임시 저장되며 전송 성공 시 삭제된다.

## 1.2.0~1.2.1: 장바구니 바로 담기 안정화

- 도입 직후 쓰던 상품 카드 전체 호버 프리패치와 iframe 3개 캐시는 같은 1.2.0 라인에서 폐기됐다.
- 현재는 담기 버튼 위 160ms 워밍, iframe 한 개, 요청별 폐기를 사용한다.
- CSP에 막히던 인라인 주입은 외부 파일 `extension/page-cart-hook.js`를 MAIN world에 주입하는 방식으로 교체됐다.
- 헤더 장바구니 숫자와 타임아웃 성공 추정은 제거됐다. 현재는 요청 응답과 확인된 완료 UI만 성공으로 인정한다.

## 1.1.0: 정렬 상태 복원·표시 항목 모델

- 정렬 상태를 같은 검색어 리로드 후 복원할 수 있도록 `chrome.storage.local`에 둔다. 현행 키는 `altActiveSort`와 `altSortQuery`다.
- 사용자 셀렉터·다중 프리셋·플로팅 피커·캡처·`activeTab`은 제거됐다. 현재는 `extension/preset-data.js` 고정 프리셋과 `altPresetOff`만 사용한다. 구 데이터는 읽거나 변환하지 않는다.

## 키워드 필터 레거시 읽기

현재 검색어 식별 키는 `excludedKeywordsForQuery`다. 과거 `excludedKeywordsSessionKey` 값은 기존 이용자의 검색어를 한 번 해석하기 위한 읽기 폴백으로만 남아 있다. 새 저장은 `excludedKeywordsForQuery`로만 한다.

제외 키워드는 검색어 `q` 변경 또는 검색 폼 재제출 때 초기화한다. 페이지·정렬·검색 개수·쿠팡 필터만 바뀌면 유지한다.

## 호환성 확인 체크리스트

1. `extension/manifest.json`과 [`버전.md`](버전.md)의 버전이 같은지 확인한다.
2. `extension/settings-defaults.js`의 `SETTINGS_VERSION`과 `DEFAULT_SETTINGS.settingsVersion`이 같은지 확인한다.
3. 설정 키를 추가·개명·제거하면 병합·변환·팝업·콘텐츠 스크립트를 함께 확인하고, 필요하면 릴리스 전에 고지한다.
4. 권한·호스트가 늘어나면 기능 범위와 개인정보처리방침을 함께 검토한다.
