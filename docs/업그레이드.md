# 업그레이드

승인된 deprecated/legacy 교체 이력. 상세 맥락은 [`방법론.md`](방법론.md), 버전 한 줄 요약은 [`버전.md`](버전.md).

## v1.16.2 — 웹 의견 작성 제거·목록 UX 정합

- **폐기:** 소개 웹의 의견 작성 폼·POST UI, 제한 문구 `오늘 2개까지`, 등록 직후 offset 미갱신, 말줄임 1회 계산, `더 보기` 연속 클릭 허용.
- **대체:** 웹은 최근 의견 조회만, 확장은 팝업에서만 제출. 제한 문구 `하루에 2개까지 보낼 수 있습니다`, 등록 시 offset +1, resize/fonts 재측정, 로딩 중 `더 보기` 잠금.

## v1.16.1 — 의견 목록 3줄·5건

- **폐기:** 의견 본문 전체 상시 노출, 한 번에 10건 로드, 버튼 문구 `더보기`.
- **대체:** 기본 3줄(`-webkit-line-clamp`) + 넘칠 때만 `펼치기`/`접기`, 페이지 크기 5, 버튼 문구 `더 보기`.

## v1.16.0 — 내부 식별자 `cra`/`addon` → `alt`

- **폐기:** `addonEnabled`, `craPresetOff`, `craActiveSort`/`craSortQuery`, DOM·메시지·CSS의 `cra`/`CRA_*`/`__cra*` 접두, `tools/cra-inspect`.
- **대체:** `altEnabled`, `altPresetOff`, `altActiveSort`/`altSortQuery`, `alt`/`ALT_*`/`__alt*` 접두, `tools/alt-inspect`. `settingsVersion` 7→8(구 키 마이그레이션 없음).

## v1.15.5 — 장바구니 성공 판정

- **폐기:** 헤더 장바구니 **숫자** 폴백; 타임아웃 시 성공 추정.
- **대체:** iframe 담기 API·DOM 문구·버튼 라벨만으로 판정. 타임아웃은 `cart_timeout` 실패.

## v1.15.4–1.15.5 — 장바구니 MAIN world 훅 주입

- **폐기:** content에서 인라인 `<script>` / `importScripts`+`func`로 PDP iframe에 훅을 심던 방식(쿠팡 `script-src` CSP에 차단).
- **대체:** `page-cart-hook.js`를 `chrome.scripting.executeScript({ world:'MAIN', files:['page-cart-hook.js'] })`로만 주입. 담기 클릭은 iframe `postMessage(__altCartCmd)`(당시 `__craCartCmd`, v1.16.0에서 리네임).

## v1.15.3 — 워밍·큐

- **폐기:** 상품 카드 호버 프리패치·캐시 최대 3개(v1.15.2); 담기 직렬 큐; 담기 후 "장바구니 보기" PDP iframe 재사용.
- **대체:** 담기 버튼 위 160ms 디바운스 워밍(슬롯 1), `pointerdown` 즉시 워밍, 담기 요청 병렬, 담기 후 iframe 폐기.

## v1.12.2 — 팝업 리로드 ↔ 정렬

- **폐기:** 팝업 설정 변경 시 `www.coupang.com` 탭 전역 `chrome.tabs.reload()`; 메모리만의 정렬 활성 플래그.
- **대체:** `storage.onChanged` 라이브 반영; `chrome.storage.local`의 정렬 복원 키(당시 `craActiveSort`/`craSortQuery` → 현행 `altActiveSort`/`altSortQuery`). 마스터(알뜰이) on/off만 리로드 유지.

## v1.7.0 — 요소 숨김 모델

- **폐기:** `hiddenElementSelectors`, `craHiddenMeta`, `craPresets`/`craActivePresetId`, 플로팅 툴(피커·드래그·캡처·undo), `captureVisibleTab`/`cra-capture`, `activeTab` 권한.
- **대체:** `preset-data.js` 고정 프리셋 + 보이기로 끈 셀렉터 목록(당시 `craPresetOff` → 현행 `altPresetOff`). UI는 팝업 하위 페이지. `settingsVersion` 4→5.
- **마이그레이션:** 구 키는 읽지 않음(storage에 남아 있어도 무해). 당시에는 표시 조정 기능이 켜진 새 사용자에게 프리셋 전체 숨김을 적용했으며, 현행 1.16.0 최초 설치에서는 표시 조정 기능 자체가 꺼져 있다.
