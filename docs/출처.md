## 초성/자모 검색 방법론: es-hangul (v1.11.0)

- **출처:** es-hangul (MIT License, Toss) — https://github.com/toss/es-hangul , https://es-hangul.slash.page
- **채택 이유:** 한국에서 초성 검색·한글 자모 처리의 사실상 표준 라이브러리(모던 API, 활발한 유지보수). v2 문서가 권장하는 검색 레시피가 명확함.
- **적용한 레시피(es-hangul v2 공식 안내):**
  - 초성 검색 = `getChoseong(word).includes(초성입력)` (예: `getChoseong('라면')==='ㄹㅁ'`).
  - 자모 부분일치(구 `hangulIncludes`) = `disassemble(x).includes(disassemble(y))` (예: `disassemble('안녕하세요')==='ㅇㅏㄴㄴㅕㅇㅎㅏㅅㅔㅇㅛ'`).
- **이식 방식:** es-hangul는 ESM 전용이고 MV3 확장은 원격 코드 로드가 금지·번들 스텝이 없으므로, 동일 알고리즘/자모 매핑만 순수 JS(`hangul-search.js`, `window.HangulSearch`)로 재구현. 겹받침(ㄳ→ㄱㅅ 등)·겹모음(ㅘ→ㅗㅏ 등)은 분해, 쌍자음(ㄲ 등)은 유지 — es-hangul 상수와 동일.
- **검색 동작:** 쿼리가 순수 초성(`^[ㄱ-ㅎ\s]+$`)이면 초성 검색, 그 외(완성형·부분 음절 포함)는 자모 분해 부분일치. 검증: `ㅅㅍ`→"상품", `상푸`→"상품", `특가`→"한정 시간 특가 상품".

| 요소 제거 대상 | 없음(하드코딩 광고 셀렉터 제거). v1.3.0부터 사용자가 직접 선택한 요소만 `hiddenElementSelectors`로 숨김 |
| 네이티브 정렬 이동 | 기존 sort `li`의 `a[href]` → `location.href` |
