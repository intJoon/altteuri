# 개인정보처리방침

> 본 문서는 대한민국 법령에 근거한 법적 고지로, 예외적으로 한국어로 작성한다.
> 「알뜰이」(이하 "본 확장")의 개인정보 처리에 관한 방침입니다.
> 시행일: 2026-07-19

## 1. 총칙 — 개인정보 처리 범위

본 확장은 개인이 비영리로 배포하는 오픈소스 브라우저 확장 프로그램입니다. 본 확장의 일반 기능은 이용자의 브라우징 기록·계정 정보·구매 내역을 수집·전송·저장하지 않으며, 해당 정보에 접근하지 않습니다.

기능 동작에 필요한 **설정값(환경설정)은 이용자 본인의 브라우저 저장소(`chrome.storage`)에만 저장**되며 개발자 운영 서비스로 전송되지 않습니다. 다만 이용자가 선택적으로 **「의견 보내기」를 사용하는 경우**, 직접 입력한 의견 내용과 확장 버전이 제품 개선을 위해 개발자 운영 서비스(`https://altteuri.vercel.app`)로 전송되어 저장됩니다.

의견 제출에 로그인·닉네임 등 개인 식별정보를 요구하거나 요청하지 않습니다. 자유 입력란에 이름, 연락처, 계정 정보 등 개인정보를 입력하지 마십시오.

## 2. 처리하는 정보 항목 및 목적

| 처리 구분 | 항목(예) | 목적 | 저장 위치 |
|-----------|----------|------|-----------|
| 기능 on/off | 알뜰이 사용 여부, 정렬·필터·페이지 표시 항목 조정·검색 개수 고정·장바구니 바로 담기 사용 여부 | 기능 활성화 상태 저장 | 이용자 브라우저 |
| 정렬 설정 | 정렬 on/off, 단위가·가격순 오름/내림, 현재 적용된 정렬 상태(local) | 정렬 상태 유지 | 이용자 브라우저 |
| 필터 설정 | 이용자가 직접 입력한 제외 키워드 | 검색결과 필터링 | 이용자 브라우저 |
| 표시 설정 | 검색 개수 고정값, 표시 항목 프리셋 해제 목록 | 화면 표시 유지 | 이용자 브라우저 |
| 의견 보내기(선택) | 이용자가 직접 입력한 의견 내용(최대 500자), 확장 버전 | 의견 목록 제공, 이용자 의견 확인 및 제품 개선 | 개발자 운영 서비스(Vercel·Neon) |
| 요청 보호 정보 | 의견 API 요청의 출발지 IP 주소, IP 주소와 UTC 일자에서 생성한 일별 HMAC 해시 | 동일 IP에서 하루 2건으로 의견 제출 제한 | 원문 IP는 API 처리 중에만 사용하며 저장하지 않음. 일별 해시는 Neon의 별도 요청 제한 테이블에 저장 |

- 브라우저 설정 항목은 특정 개인을 식별하기 위한 정보가 아니며, 이용자가 확장 설정에서 직접 입력·변경한 값입니다.
- 「제외 키워드」는 이용자가 입력한 검색 필터 문자열로, 이용자 브라우저에만 저장되고 외부로 전송되지 않습니다.
- 「의견 보내기」는 확장 팝업에서 이용자가 직접 작성·제출할 때만 전송됩니다. 소개 웹페이지에서는 최근 의견 목록만 볼 수 있습니다. 의견 내용과 확장 버전은 함께 저장되지만, 로그인·닉네임·계정 식별자는 요구하거나 별도로 결합하지 않습니다.
- 제출된 의견 내용·확장 버전·제출 시각은 확장 및 소개 웹페이지의 최근 의견 목록에서 다른 이용자에게 공개될 수 있습니다. 공개를 원하지 않는 정보나 개인정보를 입력하지 마십시오.

## 3. 저장 위치 및 보유·파기 (개인정보 보호법 제21조 참조)

- 설정값은 브라우저의 `chrome.storage.sync` 및 `chrome.storage.local`에 저장됩니다.
- `chrome.storage.sync`에 저장된 값은 이용자가 브라우저에 로그인한 본인 계정을 통해 브라우저(Google Chrome) 자체 기능으로 이용자의 다른 기기와 동기화될 수 있습니다. 이 동기화는 브라우저 제공사의 기능이며, 본 확장 개발자는 해당 데이터에 접근할 수 없습니다.
- 설정값 보유기간: 이용자가 확장을 사용하는 동안 유지됩니다.
- 설정값 파기: 이용자가 **확장 프로그램을 삭제**하거나 브라우저의 **확장/사이트 데이터 삭제** 기능을 사용하면 저장된 값은 지체 없이 삭제됩니다. (팝업에 전역 「설정 초기화」 버튼은 없으며, 검색 결과의 「전체해제」는 제외 키워드·쿠팡 필터만 초기화합니다.)
- 제출된 의견과 확장 버전은 제품 개선 목적에 필요한 기간 동안 보유하며, 목적 달성으로 더 이상 필요하지 않거나 이용자가 GitHub Issues를 통해 삭제를 요청하면 지체 없이 파기합니다.
- 파기 대상 의견은 운영 데이터베이스에서 삭제하며, 수탁자의 백업본이 존재하는 경우에는 해당 수탁자의 보존 주기에 따라 격리된 상태로 만료·삭제됩니다. 의견은 별도 식별정보를 요구하지 않으므로, 삭제 요청 시에는 대상 의견을 특정할 수 있는 내용과 제출 시점 등 필요한 정보를 함께 알려 주어야 합니다.
- 요청 제한용 일별 HMAC 해시는 원문 IP를 저장하지 않고 매일 다른 값으로 생성합니다. 7일이 지난 해시는 후속 의견 요청 처리 시 요청 제한 테이블에서 삭제합니다.

## 4. 제3자 제공 및 처리위탁

본 확장은 처리 정보를 판매하거나 광고·분석(애널리틱스)·추적 목적으로 제3자에게 제공하지 않습니다. 브라우저 설정값도 외부 사업자에게 제공하거나 처리를 위탁하지 않습니다.

「의견 보내기」 운영을 위해 다음 사업자에게 저장·인프라 처리를 위탁합니다.

| 수탁자 | 위탁 업무 | 처리 정보 |
|--------|-----------|-----------|
| Vercel Inc. | 개발자 운영 웹 서비스 및 서버리스 API 호스팅, 요청 전달 | 의견 내용, 확장 버전, 요청 처리 과정의 접속 정보 |
| Neon, Inc. | PostgreSQL 데이터베이스 호스팅 | 의견 내용, 확장 버전, 요청 제한용 일별 HMAC 해시 및 제출 횟수 |

각 수탁자는 서비스 제공에 필요한 범위에서 정보를 처리하며, 세부적인 처리 및 국외 이전 여부·보호조치는 각 사업자의 개인정보 관련 정책과 적용 법령을 따릅니다. 개발자는 위탁 업무의 목적과 범위를 벗어나 정보가 처리되지 않도록 관리합니다.

## 5. 개인정보 자동 수집 장치의 설치·운영

본 확장은 쿠키, 웹비콘 등 개인정보를 자동으로 수집하는 장치를 설치·운영하지 않습니다. 설정값 저장에는 브라우저 확장 저장소(`chrome.storage`)만 사용합니다. 다만 의견 API는 서비스 보호를 위해 요청 IP를 처리하고, 비밀키와 UTC 일자를 사용한 HMAC 해시로 변환하여 동일 IP의 의견 제출을 하루 2건으로 제한합니다. 원문 IP는 의견 데이터베이스에 저장하지 않으며, 일별 해시는 의견 본문과 분리된 요청 제한 테이블에 저장합니다.

## 6. 정보주체의 권리와 행사 방법 (개인정보 보호법 제4조 참조)

이용자는 언제든지 다음의 방법으로 처리 정보를 관리하거나 권리를 행사할 수 있습니다.

- 열람·정정: 확장 설정 팝업에서 현재 설정을 확인·변경
- 설정값 삭제·파기: 확장 프로그램 삭제, 또는 브라우저의 사이트/확장 데이터 삭제
- 제출 의견의 열람·삭제 요청: 프로젝트 저장소 GitHub Issues를 통해 요청. 별도 식별정보가 없어 대상 의견을 합리적으로 특정할 수 없는 경우에는 요청 처리가 제한될 수 있습니다.

## 7. 안전성 확보 조치

- 설정값은 개발자 운영 서비스로 전송되지 않고 이용자 브라우저 내에만 저장됩니다.
- 의견 API는 입력 길이 검증, 비정상 자동 제출 방지(honeypot), 일별 HMAC 해시 기반 동일 IP 하루 2건 제한을 적용합니다.
- 제출된 의견은 일반 문자열로 취급하며, 화면에 표시할 때 HTML로 실행하지 않고 텍스트로 렌더링하여 스크립트 삽입 위험을 줄입니다.
- 데이터베이스 자격증명은 확장에 포함하지 않고 서버리스 API 환경에서 관리합니다.
