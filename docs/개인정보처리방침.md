# 개인정보처리방침

> 본 문서는 대한민국 법령에 근거한 법적 고지다.
> 「알뜰이」(이하 "본 확장")의 개인정보 처리에 관한 방침입니다.
> 시행일: 2026-07-19

## 1. 총칙 — 개인정보 처리 범위

본 확장은 개인이 비영리로 배포하는 오픈소스 브라우저 확장 프로그램입니다. 본 확장의 일반 기능(정렬·필터·표시 조정·검색 개수·장바구니 바로 담기 등)은 이용자의 브라우징 기록·계정 정보·구매 내역을 수집·전송·저장하지 않으며, 해당 정보에 접근하지 않습니다.

기능 동작에 필요한 **설정값(환경설정)은 이용자 본인의 브라우저 확장 저장소에만 저장**되며 개발자 운영 서비스로 전송되지 않습니다.

다만 이용자가 선택적으로 **「의견 보내기」 기능**(의견 목록 보기 또는 의견 제출)을 사용하면, 요청이 개발자 운영 서비스(`https://altteuri.vercel.app`, 미국 소재 인프라)로 전달됩니다. 의견 **본문·확장 버전·요청 제한용 일별 식별값**은 **의견을 제출할 때**에만 데이터베이스에 저장됩니다. 목록을 보기만 하는 경우 본 확장은 의견 본문·식별값을 저장하지 않으나, 호스팅 사업자가 접속 과정에서 접속 정보를 처리할 수 있습니다(「5. 개인정보의 국외 이전」).

로그인·닉네임·이메일 등 **개인 식별정보를 별도로 요구하거나 수집하지 않습니다.** 그렇다고 해서 의견 기능에서 개인정보가 전혀 발생하지 않는다는 뜻은 아닙니다. 자유 입력란에 이름·연락처 등을 적으면 그 내용이 개인정보가 될 수 있고, 제출 시 처리하는 IP는 다른 정보와 결합하면 개인을 알아볼 수 있는 정보(개인정보 보호법 제2조제1호나목)에 해당할 수 있습니다. **자유 입력란에 개인정보를 입력하지 마십시오.**

## 2. 처리하는 정보 항목 및 목적 (개인정보 보호법 제15조·제22조 참조)

| 처리 구분 | 항목(예) | 목적 | 저장 위치 | 발생 시점 |
|-----------|----------|------|-----------|-----------|
| 기능 on/off | 알뜰이 사용 여부, 정렬·필터·페이지 표시 항목 조정·검색 개수 고정·장바구니 바로 담기 사용 여부 | 기능 활성화 상태 저장 | 이용자 브라우저 | 설정 변경 시 |
| 정렬 설정 | 정렬 on/off, 단위가·가격순 오름/내림, 현재 적용된 정렬 상태(local) | 정렬 상태 유지 | 이용자 브라우저 | 설정 변경 시 |
| 필터 설정 | 이용자가 직접 입력한 제외 키워드 | 검색결과 필터링 | 이용자 브라우저 | 설정 변경 시 |
| 표시 설정 | 검색 개수 고정값, 표시 항목 프리셋 해제 목록 | 화면 표시 유지 | 이용자 브라우저 | 설정 변경 시 |
| 의견 보내기(선택) | 이용자가 직접 입력한 의견 내용(최대 500자), 확장 버전 | 의견 목록 제공·공개, 이용자 의견 확인 및 제품 개선 | 개발자 운영 서비스(Vercel·Neon) | **의견 제출 시** |
| 요청 보호 정보 | 의견 제출 요청의 출발지 IP(처리 중만), IP를 복원할 수 없는 형태로 변환한 일별 식별값·제출 횟수 | 동일 IP에서 하루 2건으로 의견 제출 제한 | 원문 IP는 저장하지 않음 | **의견 제출 시** |
| 목록 보기 접속 | 의견 목록 요청에 따르는 접속 정보(호스팅 사업자 처리 가능) | 최근 의견 목록 표시 | 본 확장은 저장하지 않음. 호스팅 사업자 로그 정책에 따를 수 있음 | **목록을 보기만 할 때** |

- 브라우저 설정 항목은 특정 개인을 식별하기 위한 정보가 아니며, 이용자가 확장 설정에서 직접 입력·변경한 값입니다. 외부로 전송되지 않습니다.
- 「의견 보내기」 **작성·제출**은 확장 팝업에서만 가능합니다. 소개 웹페이지에서는 최근 의견 **목록 보기만** 가능합니다.
- 제출된 의견 내용·확장 버전·제출 시각(날짜)은 확장 및 소개 웹페이지의 최근 의견 목록에서 **다른 이용자에게 공개**될 수 있습니다. 이는 아래 「4」의 제3자 제공과 다릅니다.
- **수집·이용의 법적 근거(의견 본문·확장 버전·요청 보호 정보):** 개인정보 보호법 제15조제1항제1호(정보주체의 동의). 팝업에서 저장·공개·국외 처리 사실과 개인정보 입력 금지를 먼저 알리고, 이용자가 이를 확인한 뒤 **「보내기」를 눌러 자발적으로 제출하는 방식**으로 동의를 표시합니다. 제출하지 않아도 일반 기능 이용에는 불이익이 없습니다.
- 위 표의 처리 목적·수집 항목·보유 및 이용 기간·거부권은 같은 법 제15조제2항 고지 사항에 해당합니다.

## 3. 저장 위치 및 보유·파기 (개인정보 보호법 제21조 참조)

| 구분 | 보유 기간 | 파기 |
|------|-----------|------|
| 브라우저 설정값 | 이용자가 확장을 사용하는 동안 | 확장 삭제 또는 브라우저의 확장/사이트 데이터 삭제 시 지체 없이 삭제. (팝업에 전역 「설정 초기화」 버튼은 없으며, 검색 결과의 「전체해제」는 제외 키워드·쿠팡 필터만 초기화) |
| 제출된 의견·확장 버전 | **제출일로부터 최대 2년**, 또는 이용자가 GitHub Issues로 삭제를 요청한 때 중 **먼저 도래하는 때**까지 | 의견 API 요청 처리 시 제출일로부터 730일이 지난 행을 DB에서 삭제. 수탁자 백업본이 있으면 해당 수탁자 보존 주기에 따라 격리 후 만료·삭제. 삭제 요청 시에도 지체 없이 삭제 |
| 요청 제한용 일별 식별값 | **생성일로부터 7일** | 후속 의견 제출 처리 시 7일이 지난 레코드 삭제. 원문 IP는 저장하지 않음 |

- 설정값은 브라우저 확장 저장소에 저장됩니다.
- 설정값 중 일부는 이용자가 브라우저에 로그인한 본인 계정을 통해 브라우저(Google Chrome) 자체 기능으로 다른 기기와 동기화될 수 있습니다. 이 동기화는 브라우저 제공사의 기능이며, 본 확장 개발자는 해당 데이터에 접근할 수 없습니다. 동기화에 따른 국외 처리 여부는 브라우저 제공사의 정책을 따릅니다.
- 의견은 별도 계정·식별자를 두지 않으므로, 삭제 요청 시에는 대상 의견을 특정할 수 있는 **내용·제출 날짜** 등 필요한 정보를 함께 알려 주어야 합니다. 합리적으로 특정할 수 없으면 처리가 제한될 수 있습니다.

## 4. 제3자 제공·공개·처리위탁 (개인정보 보호법 제17조·제26조 참조)

각 수탁자는 위탁 목적 범위에서 정보를 처리하며, 개발자는 위탁 목적·범위를 벗어난 처리를 하지 않도록 관리합니다. 수탁자와의 관계는 각 사업자의 **서비스 약관 및 데이터 처리 계약(DPA)**에 따릅니다.

## 5. 개인정보의 국외 이전 (개인정보 보호법 제28조의8, 같은 법 시행령 제31조제1항제2호·제29조의10 참조)

수탁자(Vercel Inc., Neon, Inc.)는 국외에 설립·소재한 사업자이므로, 「의견 보내기」 관련 요청·저장은 국외(미국) 인프라를 거칩니다. 국외 이전의 근거와 관련 사항을 아래에 공개합니다.

**국외 이전의 근거:** 개인정보 보호법 **제28조의8제1항제3호가목**입니다. 이용자가 요청한 의견 목록 보기·제출 기능을 제공하려면 해외 클라우드의 처리위탁·보관이 필요하므로, 관련 사항을 이 방침에 공개합니다. 팝업에도 저장·공개·국외 처리 사실을 짧게 먼저 알립니다.

**목록 보기와 제출의 차이**

- **목록을 보기만 할 때:** 요청이 미국 소재 호스팅으로 전달되며, 호스팅 사업자가 접속 정보를 처리할 수 있습니다. 본 확장은 이 경우 의견 본문·식별값을 **저장하지 않습니다.**
- **의견을 제출할 때:** 본문·확장 버전·일별 식별값이 **저장**됩니다.

일반 기능(정렬·필터·표시 조정 등)만 사용하고 의견 화면을 열지 않으면, 본 확장에 의한 의견 관련 국외 요청은 발생하지 않습니다.

## 6. 개인정보 자동 수집 장치의 설치·운영

본 확장은 쿠키, 웹비콘 등 개인정보를 자동으로 수집하는 장치를 설치·운영하지 않습니다. 설정값 저장에는 브라우저 확장 저장소만 사용합니다. 의견을 제출할 때는 서비스 보호를 위해 요청 IP를 처리 중에만 사용하고, IP를 복원할 수 없는 형태로 변환한 일별 식별값으로 동일 IP 제출을 하루 2건으로 제한합니다. 원문 IP는 저장하지 않습니다.

## 7. 정보주체의 권리와 행사 방법 (개인정보 보호법 제4조 참조)

- 열람·정정: 확장 설정 팝업에서 현재 설정을 확인·변경
- 설정값 삭제·파기: 확장 프로그램 삭제, 또는 브라우저의 사이트/확장 데이터 삭제
- 동의 철회: 앞으로의 제출을 하지 않으면 됩니다. 이미 제출·공개된 의견의 삭제는 GitHub Issues로 요청
- 제출 의견의 열람·삭제 요청: GitHub Issues. 계정 식별자가 없어 대상 의견을 합리적으로 특정할 수 없으면 처리가 제한될 수 있습니다

## 8. 안전성 확보 조치 (개인정보 보호법 제29조, 같은 법 시행령 제30조·제31조제1항제3호 참조)

- 설정값은 개발자 운영 서비스로 전송되지 않고 이용자 브라우저 내에만 저장됩니다.
- 의견·요청 정보는 전송 시 HTTPS로 암호화합니다.
- 의견 제출 시 입력값을 검증하고, 동일 IP의 제출을 하루 2건으로 제한합니다.
- 요청 제한에 사용하는 원문 IP는 저장하지 않습니다.
